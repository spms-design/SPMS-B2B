# 小白部署和数据同步教程

这份教程分三段：先上传 GitHub，再连接 Vercel 公开访问，最后配置 Firebase 实时保存，并用 Google Apps Script 备份到 Google Drive 里的 Google Sheets。

## 0. 你现在手里的项目已经符合什么

这个文件夹已经是标准 Vite React 项目。Vercel 官方支持 Vite 项目部署，并且 Vite 在 Vercel 上读取浏览器端环境变量时需要 `VITE_` 前缀。Firebase 的 Firestore 官方支持 `onSnapshot` 实时监听，所以别人打开网页后能看到你修改后的数据。Google Apps Script Web App 官方要求脚本里有 `doGet` 或 `doPost`，部署后可以拿到一个 Web App URL。

参考官方文档：

- Vercel Vite: https://vercel.com/docs/frameworks/frontend/vite
- Firebase Firestore realtime: https://firebase.google.com/docs/firestore/query-data/listen
- Google Apps Script Web Apps: https://developers.google.com/apps-script/guides/web
- Apps Script deployments: https://developers.google.com/apps-script/concepts/deployments

## 1. 上传到 GitHub

1. 打开 https://github.com 并登录。
2. 右上角点 `+`，选择 `New repository`。
3. Repository name 可以填 `SPMS-B2B`。
4. 选择 `Private` 或 `Public` 都可以。
5. 不要勾选自动创建 README，因为这个文件夹里已经有 `README.md`。
6. 创建仓库后，GitHub 会显示上传方法。

如果你想用网页上传：

1. 进入刚创建的仓库。
2. 点 `uploading an existing file`。
3. 把本文件夹里的所有文件拖进去。
4. 不要拖 `node_modules`、`dist`、`.env`，这些已经写在 `.gitignore` 里。
5. 点 `Commit changes`。

如果你想用命令上传，在这个文件夹中运行：

```bash
git init
git add .
git commit -m "Initial SPMS portal package"
git branch -M main
git remote add origin 你的GitHub仓库地址
git push -u origin main
```

## 2. 连接 Vercel，让大家都能打开网页

1. 打开 https://vercel.com 并用 GitHub 登录。
2. 点 `Add New...`，选择 `Project`。
3. 找到刚上传的 GitHub 仓库，点 `Import`。
4. Framework Preset 选择 `Vite`。
5. Build Command 填 `npm run build`。
6. Output Directory 填 `dist`。
7. 第一次可以先不填环境变量，直接点 `Deploy`。
8. 部署完成后，Vercel 会给你一个网址。项目名填 `SPMS-B2B` 时，Vercel 通常会生成类似 `https://spms-b2b.vercel.app` 的网址。

这时网页已经能公开访问，但还只是本地浏览器存储。要让所有人实时同步，继续做第 3 步。

## 3. 配置 Firebase 实时保存

### 3.1 创建 Firebase 项目

1. 打开 https://console.firebase.google.com 。
2. 点 `Add project`。
3. 项目名称可以填 `SPMS-B2B`。
4. Google Analytics 可以先关闭，少一步设置。
5. 创建完成后进入项目。

### 3.2 创建网页 App，拿到配置

1. 在 Firebase 项目首页，点 Web 图标 `</>`。
2. App nickname 填 `spms-web`。
3. 不用勾 Firebase Hosting，因为我们用 Vercel。
4. 点 `Register app`。
5. Firebase 会显示一段 `firebaseConfig`，里面有这些值：

```js
apiKey
authDomain
projectId
storageBucket
messagingSenderId
appId
```

### 3.3 开启 Firestore

1. 左侧菜单进入 `Build` -> `Firestore Database`。
2. 点 `Create database`。
3. 位置选离你用户近的区域即可。
4. 规则先用测试/公开规则，方便确认功能。

Firestore Rules 填：

```txt
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    match /spms_portal/{docId} {
      allow read, write: if true;
    }
  }
}
```

### 3.4 开启 Storage，用来保存上传图片

1. 左侧菜单进入 `Build` -> `Storage`。
2. 点 `Get started`。
3. 创建完成后进入 `Rules`。

Storage Rules 填：

```txt
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    match /portal-images/{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```

### 3.5 把 Firebase 配置填进 Vercel

1. 回到 Vercel 项目。
2. 进入 `Settings` -> `Environment Variables`。
3. 按 `.env.example` 一个个新增：

```txt
VITE_FIREBASE_API_KEY=你的 apiKey
VITE_FIREBASE_AUTH_DOMAIN=你的 authDomain
VITE_FIREBASE_PROJECT_ID=你的 projectId
VITE_FIREBASE_STORAGE_BUCKET=你的 storageBucket
VITE_FIREBASE_MESSAGING_SENDER_ID=你的 messagingSenderId
VITE_FIREBASE_APP_ID=你的 appId
VITE_FIREBASE_COLLECTION=spms_portal
VITE_FIREBASE_PORTAL_DOC_ID=main
```

4. 保存后进入 `Deployments`。
5. 找到最新部署，点右侧三个点，选择 `Redeploy`。

重新部署完成后，用两台电脑或两个浏览器打开网页。一个人新增客户或修改状态，另一个页面会自动看到数据变化。

## 4. 配置 Google Apps Script 备份到 Google Drive

这里的作用是：Firebase 负责实时同步，Apps Script 负责把每次保存的完整快照追加写进 Google Sheets，作为备份记录。

### 4.1 创建 Google Sheet

1. 打开 https://drive.google.com 。
2. 新建一个 Google Sheets。
3. 文件名可以叫 `SPMS Portal Backup`。

### 4.2 粘贴 Apps Script

1. 在这个 Google Sheets 里点 `Extensions` -> `Apps Script`。
2. 删除默认代码。
3. 打开本项目里的 `google-apps-script/Code.gs`。
4. 把里面的全部代码复制到 Apps Script 编辑器。
5. 点保存。

### 4.3 部署为 Web App

1. Apps Script 右上角点 `Deploy` -> `New deployment`。
2. Select type 选择 `Web app`。
3. Description 可以填 `SPMS backup endpoint`。
4. Execute as 选择 `Me`。
5. Who has access 选择 `Anyone`。
6. 点 `Deploy`。
7. 第一次会让你授权，按提示允许。
8. 复制生成的 Web app URL。

### 4.4 填进 Vercel

1. 回到 Vercel 项目。
2. 进入 `Settings` -> `Environment Variables`。
3. 新增：

```txt
VITE_APPS_SCRIPT_WEB_APP_URL=你的 Apps Script Web app URL
```

4. 再次 `Redeploy`。

之后网页每次保存数据，Apps Script 会在 Google Sheets 的 `portal_snapshots` 工作表里追加一行备份。

## 5. 常见问题

### 网页能打开，但别人看不到我的修改

通常是 Firebase 环境变量没填、拼错，或 Vercel 改完变量后没有重新部署。检查 Vercel 的 `Environment Variables`，然后 `Redeploy`。

### 上传图片失败

通常是 Storage 没开启，或 Storage Rules 没放开 `portal-images` 路径。回 Firebase Storage 检查规则。

### Apps Script 没写入 Google Sheets

检查三点：

1. Vercel 里是否填了 `VITE_APPS_SCRIPT_WEB_APP_URL`。
2. Apps Script 是否部署为 `Web app`。
3. `Who has access` 是否是 `Anyone`。

### 这些公开规则安全吗

不适合真实敏感数据。公开规则意味着知道网页地址的人就可能读写数据。确认流程没问题后，建议下一步加登录功能和更严格的 Firestore / Storage Rules。
